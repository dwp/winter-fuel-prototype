Accessibility statement for Winter Fuel Payment

This accessibility statement applies to Winter Fuel Payment, an agent-facing website run by the Department of Work and Pensions (DWP).

Using this service

We want as many people as possible to be able to use this service. For example, that means you should be able to:

  • change colours, contrast levels and fonts
  • zoom in most content up to 300% without the text spilling off the screen
  • navigate most of the website using just a keyboard
  • navigate most of the website using speech recognition software
  • listen to most of the website using a screen reader

How accessible this service is

We know some parts of this website are not fully accessible:

  • you cannot modify the line height or spacing of text
  • the country field, on some of our address capture pages, is difficult to navigate using just a keyboard
  • on one page, you cannot skip to the main content when using a screen reader
  • some content requires horizontal scrolling when zooming in at around 300%
  • where a status message is displayed on screen, for example upon task completion, it is not read by the screen reader

What to do if you cannot access parts of this service

Use TechNow to raise any issues.

Reporting accessibility problems with this service

We’re always looking to improve the accessibility of this website. If you find any problems not listed on this page or think we’re not meeting accessibility requirements, please reach out to a member of the Seasonal Payments development team.

Enforcement procedure

The Equality and Human Rights Commission (EHRC) is responsible for enforcing the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018 (the ‘accessibility regulations’). If you’re not happy with how we respond to your complaint, contact the Equality Advisory and Support Service (EASS).

Technical information about this website’s accessibility

The Department for Work and Pensions is committed to making its website accessible, in accordance with the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018.

Compliance status

This website is partially compliant with the Web Content Accessibility Guidelines version 2.1 AA standard, due to the non-compliances listed below.

Non-accessible content

The content listed below is non-accessible for the following reasons.

Non-compliance with the accessibility regulations

On a number of pages, we have form controls that do not have either of the HTML attributes aria-labelledby or aria-label, which fails WCAG success criterion 1.3.1 and 3.3.2.

Though the label is visible, the WAVE browser plugin reports a missing label on the country field, due to the structure of the page. The impact on the user is that when navigating using the keyboard, several keypresses are needed to move from to the country field from its label. This fails WCAG success criterion 1.3.2.

WCAG success criterion 1.3.5 is failed due to a badly formatted autocomplete attribute on a number of date input fields.

There are some minimum colour contrast violations, failing WCAG success criterion 1.4.3 and 1.4.11. However, these are GOV.UK approved colours.

The Payment overview page has an issue whereby the Change link is floated right, and above content, when zooming at higher magnification (300% during testing). This fails WCAG success criterion 1.4.10.

At least one page does not have a ‘Skip to main content’ link, failing WCAG success criterion 2.4.1.

Numerous pages have a back link outside of the main content, which fails WCAG success criterion 3.2.3 and 3.2.4.

WCAG success criterion 4.1.1 is failed due to a number of HTML validation errors throughout the website.

Two issues fail WCAG success criterion 4.1.2. Throughout the International New Claims section of the website, there is an aria-expanded attribute usage that is out of place, or not in line with spec. And both the UK and International Claims sections have more than one contentinfo.

Some recently added pages could not be tested for accessibility due to ongoing development required for the go-live date of 7 October 2020. We plan to test these pages and fix the issues listed above, once the Winter Fuel Payment website goes live.

What we’re doing to improve accessibility

We will continue to iterate the design of the service and continue to test the service against the WCAG 2.1 standard.

Preparation of this accessibility statement

This statement was prepared on 18 September 2020.
